# 📖 AIRLAB Admin Panel - User Guide

## Welcome to the AIRLAB Admin Panel!

This guide will help you manage content on the AIRLAB website easily and efficiently.

---

## 🔐 Logging In

1. Navigate to: **https://airlab.unilag.edu.ng/admin-air-airlabalaba**
2. Enter your admin password
3. Click "Login"

**Note:** Contact the system administrator if you've forgotten your password.

---

After logging in, you'll see the dashboard with:

- **Statistics Cards** - Quick overview of all content
- **Quick Actions** - Buttons to add new content
- **Content Summary** - Detailed breakdown
- **Getting Started Guide** - Helpful tips

## 🎯 Managing Content

### Projects

#### Adding a New Project
1. Click "Add New Project" or navigate to Projects → "New Project"
2. Fill in the form:
   - **Title** (required) - Project name
   - **Description** (required) - Brief overview
   - **Abstract** (optional) - Detailed description
   - **Image URL** (required) - Link to project image
   - **Image Hint** (optional) - Description for accessibility
   - **Tags** - Keywords (press Enter after each tag)
   - **Status** - Ongoing, Completed, or Research Phase
   - **Link** (optional) - Project website or GitHub
   - **Paper URL** (optional) - Link to research paper
3. Click "Create Project"

#### Editing a Project
1. Go to Projects management page
2. Find the project you want to edit
3. Click "Edit" button
4. Update the information
5. Click "Update Project"

#### Deleting a Project
1. Go to Projects management page
2. Find the project
3. Click "Delete" button
4. Confirm deletion

---

### Research Papers

#### Adding a Research Paper
1. Click "Upload Research Paper"
2. Fill in the form:
   - **Title** (required) - Paper title
   - **Authors** (required) - All authors (comma-separated)
   - **Year** (required) - Publication year
   - **Description** (required) - Brief summary
   - **Abstract** (optional) - Full abstract
   - **File URL** (required) - Link to PDF
   - **Image URL** (optional) - Cover image
3. Click "Create Research Paper"

#### Editing/Deleting
- Same process as Projects

---

### Team Members

#### Adding a Team Member
1. Click "Add Team Member"
2. Select category:
   - **Leading** - Lab leaders and directors
   - **Pioneer** - Core team members
   - **Volunteers** - Student volunteers
3. Fill in the form:
   - **Name** (required)
   - **Role** (required) - Position/title
   - **Image URL** (required) - Profile photo
   - **Image Hint** (optional) - Photo description
   - **Bio** (required) - Background and expertise
   - **Social Links** (optional):
     - LinkedIn
     - Twitter/X
     - GitHub
     - Email
4. Click "Create Team Member"

#### Editing/Deleting
- Same process as Projects
- Remember to select the correct category

---

### News & Events

#### Adding News
1. Click "Create News/Event"
2. Fill in the form:
   - **Title** (required) - News headline
   - **Date** (required) - Publication date
   - **Link** (required) - Full article URL
3. Click "Create News"

**Note:** News items appear in reverse chronological order (newest first)

---

### History Timeline

#### Adding a History Item
1. Click "Add History Item"
2. Fill in the form:
   - **Year** (required) - Event year
   - **Event** (required) - Event title
   - **Description** (required) - Event details
   - **Image** (optional) - Event photo
   - **Link** (optional) - More information URL
3. Click "Create History Item"

**Note:** History items are automatically sorted by year

## 💡 Best Practices

### Images
- **Use high-quality images** (minimum 800x600px)
- **Optimize before uploading** (use tools like TinyPNG)
- **Use descriptive image hints** for accessibility
- **Recommended sources:**
  - Unsplash (https://unsplash.com)
  - Your own photos
  - Lab event photos

### Writing Content
- **Be concise** - Keep descriptions brief and clear
- **Use proper grammar** - Proofread before publishing
- **Include keywords** - Help with search and discovery
- **Update regularly** - Keep content fresh and relevant

### Tags
- Use relevant, specific tags
- Keep tags consistent across similar projects
- Examples: "Machine Learning", "Robotics", "Computer Vision"

### Links
- Always use full URLs (https://...)
- Test links before publishing
- Use reputable sources
- Keep links up to date

## ⚠️ Common Issues

### "Validation failed" Error
- Check all required fields are filled
- Ensure URLs are valid (start with https://)
- Verify email addresses are correct
- Check year is a valid number

### Image Not Showing
- Verify the image URL is correct
- Ensure the image is publicly accessible
- Try a different image URL
- Check image format (JPG, PNG, WebP supported)

### Changes Not Appearing
- Refresh the public website page
- Clear your browser cache
- Wait a few seconds for updates to process

### Can't Login
- Verify password is correct (case-sensitive)
- Clear browser cookies
- Try incognito/private mode
- Contact administrator if issue persists
